About Git

По умолчанию в каждом проекте git создается ветка master.

Чтобы посмотреть какие ветки есть в репозитории используется команда: git branch
Звездочка перед именем ветки указывает, что текущая разработка ведется в этой ветки.

Для создания новой ветки используется команда: git branch имя_новой_ветки

Чтобы переключить разработку в новую ветку нужно использовать команду: git checkout имя_новой_ветки
Теперь, если вызвать команду git branch, то увидим звездочку на новой ветке

Теперь после работы в новой ветке можно вызвать команду git status
Команда git status показывает все новые файлы и изменившиеся файлы и их нужно отметить для внесения в репозиторий.

Для этого выполняем команду git add имя_нового_файла
Для этого выполняем команду git add имя_изменившегося_файла

Теперь зафиксируем эти изменения в локальном репозитории командой git commit -m "New_Commit_Message"
Теперь нужно добавить изменения на удаленный репозиторий: git push origin имя_новой_ветки
Эта команда создаст удаленную ветку имя_новой_ветки и внесет изменения в гитхабе.

Чтобы вернуться в ветку master нужна команда: git checkout master

В ветке master всех новых файлов и изменений старых файлов - не будет, что в принципе и логично.
